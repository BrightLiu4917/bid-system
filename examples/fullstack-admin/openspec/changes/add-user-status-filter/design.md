# Design

## Product and UX
- 入口：用户列表页。
- 主流程：进入列表，展开或查看筛选区，选择状态，列表刷新。
- 空态：筛选无结果时显示筛选条件无匹配，不显示“无权限”。
- 错误态：状态选项加载失败时保留列表可用，并提供重试。

## API
- 状态选项接口：待确认是否已有。
- 用户列表接口：如新增 `status` 查询参数，必须保持向后兼容。

## Data
- 不修改表结构。
- 不新增状态枚举。

## Frontend
- 状态选项从 API 或已确认常量读取。
- 筛选条件进入 URL query 或既有列表状态管理，遵循项目约定。
- 加载态、错误态和空态必须覆盖。

## Backend
- 如已有状态枚举，后端返回 label/value。
- 如列表接口已有筛选参数，仅做契约确认。

## Rollback
前端移除筛选控件并停止传参；如新增状态选项接口，可回滚接口代码。
