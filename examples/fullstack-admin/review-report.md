# Review Report: User Status Filter

## 阻塞问题
- 无。示例保留状态枚举来源为待确认，未直接实现未确认枚举。

## 重要问题
- 如果用户列表接口尚不支持状态筛选，需要补 API 契约并确认兼容性。
- 状态选项是否受租户影响尚未确认，实现前必须澄清。

## 次要建议
- 状态筛选条件建议进入 URL query，方便刷新和分享；最终以项目既有列表状态管理约定为准。

## 表结构/字段变化审查
- 无表结构变化。

## API/前端契约审查
- 状态选项接口为新增接口。
- 列表筛选参数是否新增待确认。

## SQL 性能风险审查
- 如果按状态筛选用户列表，需要确认状态字段与租户、软删除、排序字段的索引组合。

## 安全和权限审查
- 状态选项接口必须校验用户列表查看权限。

## 验证步骤
- 用户列表默认加载。
- 状态选项加载成功和失败。
- 状态筛选、清空筛选、筛选无结果。
- 无权限用户访问。

## 风险总结
- 主要风险是枚举来源和租户影响未确认，因此实现前必须回到 OpenSpec 追问。
