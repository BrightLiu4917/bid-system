# 项目上下文

本文件记录 `zcc-bidding-system` 的业务语言和待确认概念。它不是某个具体需求的实现说明；具体功能必须进入 `openspec/changes/<change-id>/` 确认。

## 业务领域
当前项目方向为招投标业务系统。以下术语用于对齐沟通语言。标记为“已确认”的内容可作为高层业务背景；字段、状态、权限、接口和数据结构仍必须通过 OpenSpec change 细化后才能实现。

## 已确认角色

**平台管理**:
负责审核专家和供应商入驻资料，发布招标公告，审核供应商对招标公告的申请。
_状态_: 已确认。

**专家**:
通过注册、资料提交进入入驻流程，由平台管理审核。专家入驻后的具体业务职责仍待确认。
_状态_: 已确认。

**供应商**:
通过注册、资料提交进入入驻流程，由平台管理审核；可查看平台发布的招标公告并发起申请，申请由平台管理审核。
_状态_: 已确认。

## 已确认业务流程

**入驻流程**:
专家和供应商注册后提交资料，平台管理审核资料后决定是否允许入驻。
_状态_: 已确认高层流程，资料字段、审核状态和驳回重提规则待确认。

**招标公告发布流程**:
平台管理发布招标公告，供应商可查看公告。
_状态_: 已确认高层流程，公告字段、状态和展示范围待确认。

**供应商申请流程**:
供应商查看招标公告后可以发起申请，平台管理对供应商申请进行审核。
_状态_: 已确认高层流程，申请字段、审核状态和后续业务动作待确认。

## 业务术语

**报名**:
供应商对招标公告发起申请的动作是否等同于报名，仍待确认。
_状态_: 待确认术语。

**招标公告**:
平台管理发布、供应商可查看并申请的招标信息。
_状态_: 已确认高层概念，字段和状态待确认。

**入驻资料**:
专家或供应商注册后提交给平台管理审核的资料。
_状态_: 已确认高层概念，字段待确认。

**申请**:
供应商针对招标公告发起参与请求，并由平台管理审核。
_状态_: 已确认高层概念，字段和状态待确认。

**投标材料**:
供应商申请通过后是否需要提交投标材料，仍待确认。
_状态_: 待确认。

**开标**:
到达投标截止时间后，对投标信息进行开启或进入评审的环节。
_状态_: 待确认。

**评审**:
评审人员或评审流程对投标材料进行审核、评分、排序或给出意见的过程。
_状态_: 待确认。

**中标结果**:
招标项目最终确定的中标方、结果说明和公示信息。
_状态_: 待确认。

**废标**:
招标项目因规则或业务原因终止或结果无效。
_状态_: 待确认。

## 状态候选

以下状态仅作为建模候选，不能直接实现：

**入驻审核状态候选**:
- 待提交
- 待审核
- 已通过
- 已驳回

**招标公告状态候选**:
- 草稿
- 已发布
- 已关闭

**供应商申请状态候选**:
- 待审核
- 已通过
- 已驳回
- 已撤回

**投标材料状态候选**:
- 未提交
- 已提交
- 已撤回
- 已作废

## 已确认技术上下文

- 后端目录：`api/`
- 前端目录：`vue/`
- 后端基础：Java 17、Spring Boot 4、Maven Wrapper。
- 前端基础：Vue 3、Element Plus、pnpm、Rspack。
- 本地 Agent：OpenCode + qwen3-coder。
- 统一验证入口：`bash scripts/run-tests.sh`。

## 必须确认后才能实现的规则
- 角色和权限边界。
- 租户和数据隔离规则。
- 专家和供应商入驻资料字段、审核状态和驳回重提规则。
- 招标公告字段、发布规则、可见范围和状态流转。
- 供应商申请字段、审核状态和后续动作。
- 供应商申请是否等同于报名。
- 投标、开标、评审、中标、公示、废标是否在当前范围内，以及对应时间规则。
- 附件、标书、签章、加密和文件权限规则。
- 专家是否参与评审，以及评审方式和评分规则。
- 通知规则。
- 删除、撤回、作废和归档规则。
- API 统一响应格式、错误码、分页格式和鉴权方式。
- MySQL 表结构、字段含义、索引和迁移 SQL。
