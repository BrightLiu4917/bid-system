# 新项目快速接入

## 方式一：交互式安装

适合第一次使用：

```bash
bash scripts/setup-control-system.sh
```

脚本会询问：

- 项目目录
- 项目名称
- 是否全栈
- 是否 Spring Boot
- 是否 Vue / React
- 是否 gupo 项目
- 是否启用 OpenCode/qwen-coder
- 是否启用 deepv4

## 方式二：非交互式安装

适合你已经知道项目目录和参数：

```bash
bash scripts/bootstrap-new-project.sh \
  --project-dir /path/to/project \
  --project-name my-project \
  --profile local-agent \
  --local-agent \
  --backend api \
  --frontend vue
```

## 常见组合

### 普通项目

```bash
bash scripts/bootstrap-new-project.sh \
  --project-dir /path/to/project \
  --profile default
```

### api + vue 全栈项目

```bash
bash scripts/bootstrap-new-project.sh \
  --project-dir /path/to/project \
  --project-name bidding-system \
  --profile local-agent \
  --local-agent \
  --backend api \
  --frontend vue
```

### gupo 项目

```bash
bash scripts/bootstrap-new-project.sh \
  --project-dir /path/to/gupo-project \
  --profile gupo \
  --gupo
```

### 启用 deepv4

```bash
bash scripts/bootstrap-new-project.sh \
  --project-dir /path/to/project \
  --profile local-agent \
  --local-agent \
  --deepv4
```

安装后编辑：

```text
.agent/local-agent.env
```

填入真实：

```text
DEEPV4_BASE_URL
DEEPV4_API_KEY
DEEPV4_MODEL
```

## 安装后必须检查

```bash
bash scripts/skill-check.sh
bash scripts/route-check.sh
bash scripts/docs-link-check.sh
```

## 第一个需求怎么做

1. 创建 OpenSpec change：

```bash
mkdir -p openspec/changes/add-my-feature/specs/my-capability
cp templates/openspec-change/proposal.md openspec/changes/add-my-feature/proposal.md
cp templates/openspec-change/design.md openspec/changes/add-my-feature/design.md
cp templates/openspec-change/tasks.md openspec/changes/add-my-feature/tasks.md
```

2. 在 Codex 里说：

```text
读取 openspec/changes/add-my-feature，帮我做需求预审、架构设计、表结构/API/任务拆分。
```

3. 需求确认后再进入实现。

## 注意事项

- 空项目建议先 `git init`。
- 不清楚字段、状态、权限、API 时不要让 qwen 直接写代码。
- 数据库变更必须先 DBA 设计并确认。
- `.agent/local-agent.env` 不要提交。
