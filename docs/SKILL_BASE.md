# Skill 公共模板

本文件是新增或维护 `.codex/skills/<skill-name>/SKILL.md` 时的公共基线，不是 Codex 自动发现入口。

## Frontmatter

```yaml
---
name: <skill-name>
description: <中文描述，包含触发场景、职责边界、必要工具或关键约束>
---
```

要求：
- `name` 必须与目录名完全一致。
- `description` 用中文写清楚“什么时候使用”和“不能替代什么”。
- 路径、命令、技术名保留英文。

## 标准章节

建议按以下顺序组织：

```md
# <标题>

## 适用场景
- ...

## 必须读取
- `AGENTS.md`
- `docs/SKILL_ROUTING.md`
- ...

## 流程
1. ...

## 停止并询问
- ...

## 禁止
- ...

## 输出要求
- ...
```

## 公共规则
- 只保留本 skill 执行所需的核心规则，不把所有全局规则复制进来。
- 详细工程规则放在 `docs/`，由 `必须读取` 引用。
- 可执行、可重复、易错的动作优先放到 `scripts/` 或 `tools/`。
- OpenSpec 相关操作只放 `workflow-*`。
- Matt Pocock 风格方法只放 `method-*`，不直接写 OpenSpec。
- 兼容入口只能指向新主入口，不承载完整实现规则。

## 停止并询问基线
出现以下情况必须停止并询问：
- 输入信息缺失会影响正确性。
- 业务规则、字段、枚举、权限、租户、状态流转或 API 响应不清。
- 需要覆盖、删除、重命名或迁移已有产物。
- 需要引入新依赖或改变架构。
- 当前 skill 与其他 skill 职责冲突且无法按 `docs/SKILL_ROUTING.md` 判定顺序。

## 输出基线
输出至少包含：
- 使用的 skill。
- 读取的事实源。
- 实施或审查结果。
- 变更文件，如有。
- 验证步骤。
- 假设和风险。
