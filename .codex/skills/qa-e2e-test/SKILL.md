# E2E 测试

## 必须读取
- `AGENTS.md`
- `docs/TESTING_RULES.md`
- 相关 OpenSpec specs/changes

## 流程
1. 从 OpenSpec 和任务目标识别核心用户路径。
2. 覆盖正常流程、权限不足、空数据、参数错误、状态不允许和失败反馈。
3. 优先使用项目既有 E2E 工具。
4. 无法自动化时，输出可执行的手动验收步骤。
5. 记录验证结果和残余风险。

## 输出
- E2E 范围
- 用例清单
- 执行命令或手动步骤
- 验证结果
- 风险
