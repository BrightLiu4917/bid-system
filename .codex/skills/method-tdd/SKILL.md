# Spring Boot 测试驱动开发

## 必须读取
- `AGENTS.md`
- `CONTEXT.md` 或 `CONTEXT-MAP.md`
- `docs/TESTING_RULES.md`
- `docs/SPRING_BOOT_RULES.md`

## 流程
1. 找到既有测试框架和运行命令。
2. 明确要验证的公开行为，不测试私有实现细节。
3. 一次只写一个行为测试，禁止一次性横向写完全部测试。
4. 先让测试失败，再写最小代码通过。
5. 每个红绿循环后再进入下一个行为。
6. 所有测试通过后再重构，重构期间保持绿色。
7. 无法写测试时，说明缺少正确测试 seam 的原因，并给出最小人工验证步骤。

## 测试原则
- 测试名称使用 `CONTEXT.md` 中的领域语言。
- 优先通过公开接口验证行为。
- 避免 mock 内部协作者，除非外部系统或不可控依赖。
- 测试应能在内部重构后仍然有效。

## 输出
- 测试范围
- 新增/修改测试
- 运行命令
- 结果
- 无法测试的 seam，如有
- 未覆盖风险
